# Miroku 機能説明書

営業支援システム「Miroku（ミロク）」の機能について、非エンジニア向けにまとめた説明書です。

---

## はじめに

### Miroku とは

**Miroku** は、営業チームの売上をリアルタイムで可視化し、チーム全体の成果を最大化するための営業支援システムです。

主な特徴：

- **リアルタイム売上ダッシュボード** — 売上を即座にグラフで確認
- **ディスプレイモード** — オフィスの大画面に自動表示してチームで情報共有
- **速報演出** — 売上登録時に祝福動画を全画面で再生し、Supabase Realtime によりほぼ遅延なく全端末へ配信
- **外部連携** — LINE や Google Chat に自動通知
- **マルチテナント** — 1 つのシステムで複数の会社が独立して利用可能
- **モバイル対応** — スマホでもそのまま使える

---

## 1. ユーザーの種類と権限

Miroku には大きく 4 種類のユーザーがあります。

| ユーザー種別                     | できること                                         | ライセンス人数   |
| -------------------------------- | -------------------------------------------------- | ---------------- |
| **スーパー管理者** (SUPER_ADMIN) | 全テナントの管理、契約管理                         | 対象外           |
| **管理者** (ADMIN)               | テナント内の全設定・全データ管理                   | 対象外           |
| **メンバー** (USER)              | データ入力、ダッシュボード閲覧、ランキング表示対象 | **カウント対象** |
| **入力担当者** (Operator)        | 他メンバーの代理データ入力                         | 対象外           |

### 権限の違い

- **スーパー管理者**: 運営側の管理者。複数の会社（テナント）を一括管理できる。`/admin` 配下にアクセス可能
- **管理者**: 自社（テナント）のすべての設定・データを管理できる。`/settings` 配下にアクセス可能
- **メンバー**: 営業担当者本人。自分の売上を入力し、ランキングに表示される
- **入力担当者**: 事務員など、他のメンバーの売上を代理で入力する人。ランキングには表示されず、ライセンス人数にもカウントされない（内部的には role=USER だが isOperator=true）

### スーパー管理者によるテナント代理ログイン

スーパー管理者は、テナントの会社アカウント（accountCode）と自分のメールアドレス・パスワードを入力することで、対象テナントの管理者として一時的にログインできます。サポート対応や障害調査などに使われ、その間のセッションには `isSuperAdminImpersonating` フラグが付きます（利用規約同意画面のスキップなど一部の動作が異なります）。

## 2. 画面一覧

### 2.1 ログイン画面

会社アカウント（テナントスラッグ）とメールアドレス・パスワードでログインします。

- 会社アカウント入力 → メール・パスワード入力の 2 段階フロー
- 会社アカウントを URL に含めた直リンクログインも可（例: `/login/acme`）
- スーパー管理者用の別ログイン（`/admin/login`）

#### ログイン失敗時の記録

不正アクセス検知のため、以下の場合は監査ログに `USER_LOGIN_FAILED` として記録されます。

- 入力不備
- 会社アカウントが見つからない
- ユーザーが見つからない
- テナントが無効化されている
- パスワードが違う

記録項目: 失敗理由、入力されたメールアドレス・会社アカウント、IP アドレス、（特定できた場合のみ）ユーザーID／テナントID

加えて、ユーザー存在判定のタイミング攻撃を緩和するため、いずれの失敗パスでも常に bcrypt 比較を 1 回実行する仕組みになっています。

---

### 2.2 ダッシュボード（トップ画面）

ログイン後に最初に表示される、売上の状況がひと目でわかる画面です（`/`）。

#### 表示切り替え

以下の 7 種類のビューを画面上部のタブで切り替えられます。

| ビュー (ViewType)                   | 内容                                                             |
| ----------------------------------- | ---------------------------------------------------------------- |
| **期間グラフ** (PERIOD_GRAPH)       | 期間内の日別売上をグラフ表示                                     |
| **累計グラフ** (CUMULATIVE_GRAPH)   | 期間累計の売上推移をグラフ表示                                   |
| **トレンド** (TREND_GRAPH)          | 月別売上の推移                                                   |
| **レポート** (REPORT)               | 日平均・月平均・曜日別分析・目標達成予測など詳細統計             |
| **ランキング** (RECORD)             | メンバー別の売上ランキング                                       |
| **数字ドン** (NUMBER_BOARD)         | 合計売上・登録件数・平均達成率・チーム目標を大きな数字で強調表示 |
| **カスタムスライド** (CUSTOM_SLIDE) | 画像・YouTube・テキストの任意スライド                            |

#### 絞り込み機能（フィルターバー）

- **グループ絞り込み** — 特定のグループだけ表示
- **メンバー絞り込み** — 特定のメンバーだけ表示
- **データ種別切り替え** — 粗利、媒介数などのデータ種別を切り替え
- **期間切り替え** — 年初〜当月（YTD）、直近 3 ヶ月、直近 6 ヶ月、会計年度、カスタム指定

#### 初期セットアップウィザード

新規テナント作成直後（`Tenant.setupCompleted = false`）には、初期設定を順に行うウィザードが表示されます。

1. **グループ作成** — 営業部、営業 1 課など
2. **メンバー登録** — 営業担当者を追加
3. **データ種別設定** — 売上、契約件数などのデータ種別を設定

完了すると `setupCompleted = true` になり、以降は表示されません。

---

### 2.3 データ管理画面

`/sales/records` でアクセスできる、売上データの一覧画面です。

主な機能:

- **一覧表示** — 全売上レコードを表形式で表示
- **詳細検索** — 日付範囲、グループ、メンバー、データ種別で絞り込み
- **編集・削除** — 個別に編集・削除可能（編集・削除のたびに監査ログ＋ディスプレイ画面の数値再取得が発火）
- **CSV / Excel インポート** — 一括登録（カスタムフィールド対応、プレビュー付き）
- **CSV エクスポート** — 絞り込み条件に合う全データをダウンロード

---

### 2.4 ディスプレイモード

`/display` でアクセスできる、オフィスの大型ディスプレイ向けの画面です。

#### 特徴

- 複数のビューを **自動で順番に切り替え**
- 各ビューの表示時間を秒単位で設定可能
- 見終わると最初に戻るループ再生（オン/オフ切り替え可能）
- **ダークモード** に切り替え可能
- 会社ロゴ・チーム名を常時表示
- **マウスカーソル自動非表示**（一定時間操作がないと消える）

#### 速報（Breaking News）機能

誰かが売上データを登録すると、全画面で **祝福演出（動画）** が再生されます。

**現在の仕組み（重要）:**

- ディスプレイ画面は **Supabase Realtime の `tenant-events-{tenantId}` チャネル** を購読
- 売上登録時、サーバ側が `new-record` イベントを broadcast
- ディスプレイ画面は ID をもとに `/api/sales/breaking-news?id=...` で表示用データ（メンバー名・金額・速報メッセージ・動画 ID）を取得して再生
- ポーリングは廃止済みで、新規登録から速報表示までの遅延はほぼゼロ

**動画の種類:**

- **動画 1** — シンプルな祝い動画
- **動画 2** — 表彰式風の演出
- **動画 3** — 豪華な演出

データ種別ごとに「速報を有効化するか」「メッセージ」「動画 ID」をカスタマイズできます。

#### データ変更通知

売上の登録・更新・削除のいずれが行われた場合も、同チャネルの `data-changed` イベントが broadcast されます。ディスプレイ画面側では debounce 付きでビューデータを再取得するため、編集・削除も即座に反映されます。

---

### 2.5 ヘルプ画面

`/help` でアクセスできる、使い方ガイドです。

- **操作ガイド** — 各機能の使い方
- **よくある質問** — FAQ
- **お問い合わせ** — サポート連絡先

## 3. 設定画面

設定画面（`/settings`）には、以下の 11 のセクションがあります（ADMIN 権限必須）。URL に `?section=xxx` を付けることで直接該当セクションを開けます。

| #   | セクション                           | URL クエリ             |
| --- | ------------------------------------ | ---------------------- |
| 1   | メンバー設定                         | `?section=member`      |
| 2   | グループ設定                         | `?section=group`       |
| 3   | データ種類管理                       | `?section=dataType`    |
| 4   | グラフ設定                           | `?section=graph`       |
| 5   | ディスプレイモード設定               | `?section=display`     |
| 6   | 目標設定                             | `?section=target`      |
| 7   | データ入力設定（カスタムフィールド） | `?section=record`      |
| 8   | システム設定                         | `?section=system`      |
| 9   | 外部連携設定                         | `?section=integration` |
| 10  | 操作ログ閲覧                         | `?section=log`         |
| 11  | 契約情報                             | `?section=license`     |

### 3.1 メンバー設定（`/settings?section=member`）

メンバー、管理者、入力担当者を 3 つのタブで管理します。

**できること:**

- 新規追加（メール・パスワード・役割・部署・プロフィール画像）
- 編集（名前・メール・部署・アイコン・有効/無効・**ロール**を変更可能）
- 削除
- パスワード変更（自分自身、または管理者から下位ロールへ）
- CSV/Excel 一括インポート（メンバー / 管理者 / 入力担当者を選択可能）

**ロール変更の制約:**

- 編集モーダルでロールを「メンバー / 管理者 / 入力担当者」の 3 択から選択
- **自分自身のロールは変更できない**（誤操作防止）
- API 側でも、ライセンス対象（USER かつ非 Operator）への昇格時にライセンス上限チェックを実施

**プロフィール変更時のセッション同期:**

自分自身を編集した場合、画面上の名前やアイコンが即座に反映されるよう、保存時に NextAuth セッションを更新します（再ログイン不要）。

---

### 3.2 グループ設定（`/settings?section=group`）

営業チームや営業所などの **グループ** を管理します。

**できること:**

- グループ作成・編集・削除
- メンバーの追加・終了・削除
- **メンバーシップの期間管理** — 開始月・終了月を設定して履歴を残せる（転勤・異動の記録）
- グループ画像のアップロード

履歴管理のおかげで、「過去の月では A さんは営業 1 課、現在は営業 2 課」といった切り替わりがあっても、各月のランキングに正しいグループ所属が反映されます。

---

### 3.3 データ種類管理（`/settings?section=dataType`）

売上データを分類する **データ種別** を管理します。会社によって異なる売上指標を、複数並行して扱えます。

**属性:**

- **名前** — 「粗利」「契約件数」「媒介数」など
- **単位** — 万円 / 千円 / 円 / 件 / 時間 / 分 / 個 / 回 / 人 から選択（内部 enum: `Unit`）
- **色** — グラフ表示時の色
- **デフォルト** — 新規登録時に最初に選ばれる種別（1 つだけ）
- **表示順** — ドラッグ操作で並び替え可
- **有効/無効** — 使わなくなった種別はアーカイブ可能

データ種別ごとに **カスタムフィールド** と **速報設定（メッセージ・動画）** を別々に持てます。

---

### 3.4 グラフ設定（`/settings?section=graph`）

ダッシュボード／ディスプレイモードのグラフの見た目をカスタマイズします。

- ランキングの配色（TOP / CENTER / LOW）
- バースタイル（円柱 / フラット / 角丸）
- 目標ライン・移動平均線の表示／非表示
- グラデーション・グロー強度（NONE / LIGHT / NORMAL / STRONG）
- ランキング表示件数の上限（空欄＝無制限）
- ダッシュボード／レポートでの表示既定値（年・月・期間）

---

### 3.5 ディスプレイモード設定（`/settings?section=display`）

ディスプレイモードの表示内容を細かく設定します。

#### ビュー設定

- 各ビュー（期間グラフ・累計グラフ・トレンド・レポート・ランキング・数字ドン・カスタムスライド）の **有効/無効**
- **表示時間**（秒）
- **表示順序**
- ビューごとのタイトル
- ビューごとの **データ種別フィルター**
- **期間プリセット** — 年初〜当月、直近 3/6 ヶ月、会計年度、カスタム

#### 再生設定

- **ループ再生** — 最後まで再生したら最初に戻るか
- **データ更新間隔** — 10 秒 / 30 秒 / 1 分 / 5 分 / 15 分 / 30 分（broadcast による即時更新と併用）
- **トランジション効果** — なし / フェード / 左スライド / 右スライド

#### フィルター設定

- 表示対象のグループ・メンバー・データ種別を絞り込み

#### 表示情報設定

- 会社ロゴ URL
- チーム名
- ダーク/ライトモード

#### 速報設定

- データ種別ごとに **有効/無効** を個別指定
- 速報メッセージ（「おめでとう！」など）
- 速報動画を 3 種類から選択（プレビュー付き）

#### カスタムスライド管理

- **画像スライド** (IMAGE) — 画像 URL を指定して表示
- **YouTube スライド** (YOUTUBE) — YouTube のリンクを貼り付け（動画が終わると次のビューへ）
- **テキストスライド** (TEXT) — 大きな文字でメッセージ表示（キャンペーン告知など）

---

### 3.6 目標設定（`/settings?section=target`）

メンバーやグループごとに **売上目標** を設定します。

**2 つのタブ:**

- **個人目標** — メンバーごとに月別目標を設定
- **グループ目標** — グループごとに月別目標を設定

**できること:**

- 年度・データ種別を切り替えて編集
- Excel のような表形式で一括入力
- **CSV/Excel インポート**（個人目標のみ対応）
  - 従業員名、目標値、年月（yyyyMM）の縦型データをインポート

目標値はダッシュボードのグラフに **目標ライン** として表示され、達成率の計算にも使われます。

---

### 3.7 データ入力設定（カスタムフィールド）（`/settings?section=record`）

売上データに会社独自の項目を追加できる機能です。

**カスタムフィールドの種類 (CustomFieldType):**

- **テキスト** (TEXT) — 自由入力（例: 顧客名、案件名）
- **日付** (DATE) — 日付ピッカー（例: 契約日、納期）
- **プルダウン** (SELECT) — 選択肢から選ぶ（例: 案件種別、営業段階）
- **数値** (NUMBER) — 数値入力。`aggregatable=true` を ON にすると単位付きで集計対象に追加可能

**特徴:**

- **データ種別ごとに異なるフィールドを設定可能**
  - 例: 粗利には「顧客名」、媒介数には「物件名」などを設定
- 必須項目／任意項目の切り替え
- 表示順序の設定
- 有効／無効切り替え（既存データを保持したまま非表示化）

---

### 3.8 システム設定（`/settings?section=system`）

システム全体の設定です。

- **会計年度の開始月**（デフォルト: 4 月）

会計年度設定は、ダッシュボードのフィルター「会計年度」やデフォルト期間表示に影響します。

---

### 3.9 外部連携設定（`/settings?section=integration`）

チャットツールとの連携を設定します。売上データ登録時に自動通知が送られます。

#### LINE Messaging API

- **Channel Access Token** と **Group ID** を入力
- テスト送信で接続確認
- 売上登録時に LINE グループに自動通知（メンバー名・金額・データ種別・カスタムフィールド値を整形）

#### Google Chat

- **Webhook URL** を貼り付け
- テスト送信で接続確認
- 売上登録時に Google Chat スペースに自動通知

両連携とも「未接続 (DISCONNECTED) / 接続済み (CONNECTED)」のステータスを持ち、CONNECTED でない場合は通知を送りません。

---

### 3.10 操作ログ閲覧（`/settings?section=log`）

すべての操作履歴（自テナント分）を閲覧できます。

**記録される操作 (AuditAction enum):**

- ログイン・**ログイン失敗**・ログアウト
- ユーザー追加・更新・削除・パスワード変更・利用規約同意
- グループの作成・更新・削除、メンバー追加・終了・削除、メンバー一括同期
- 売上データの作成・更新・削除
- 目標設定（個人・グループ・一括）
- 各種設定変更（システム、連携、ディスプレイ、グラフ）
- カスタムフィールドの作成・更新・削除
- カスタムスライドの作成・更新・削除
- データ種別の作成・更新・削除
- テナントの作成・更新・削除（スーパー管理者のみ）
- 契約の作成・更新・期限切れ

**検索機能:**

- ユーザー別
- 日付範囲
- アクション種別

**記録される情報:** 日時、ユーザー、テナント、アクション、詳細、IP アドレス（ログイン失敗時など）。

---

### 3.11 契約情報（`/settings?section=license`）

自社のライセンス情報を確認できます。

**表示内容:**

- **プラン種別** — トライアル (TRIAL) / スタンダード (STANDARD) / エンタープライズ (ENTERPRISE)
- **ライセンス期間** — 開始日〜終了日
- **メンバー数** — 現在の人数 / 上限（無制限の場合は「無制限」）
- **残り日数** — ライセンス期限までの日数
- **契約履歴** — すべての契約変更履歴

**ライセンスの状態バッジ:**

- 緑「有効」: 期限まで 8 日以上、かつトライアル以外
- 青「トライアル」: planType === TRIAL（残期間内）
- 黄「残●日」: 期限まで 7 日以下
- 赤「期限切れ」: 期限超過。データの登録・更新・削除は制限される

## 4. データ入力

### 4.1 売上データの入力方法

画面上部の **「データ入力」** ボタンをクリックするとモーダルが表示されます。

**入力項目:**

- **メンバー**（他のメンバーの代理入力も可能）
- **データ種別**（複数あれば切り替え。デフォルト指定の種別が初期選択）
- **金額／値**（単位に応じて表示と保存値の倍率を自動換算。例: 万円なら 100 と入力 → 1,000,000 で保存）
- **日付**（デフォルトは現在日時 JST）
- **備考**（任意）
- **速報通知** — チェックを外すと、その登録は速報を流さない（デフォルト ON）
- **カスタムフィールド** — データ種別ごとに異なる項目が動的に表示される

**バリデーション:**

- メイン値が 0 かつカスタムフィールドもすべて空の場合は登録できない（無意味なレコード防止）
- 必須カスタムフィールドの未入力チェック

**登録の影響:**

- ダッシュボードに反映
- ディスプレイモードに **速報** が流れる（速報通知 ON のときのみ）
- ディスプレイモードの数字／グラフが即座に再取得される（編集・削除時も同様）
- LINE / Google Chat に通知（接続済みの場合）

**対応しているインポート:**

## 5. スーパー管理者画面

`/admin` でアクセスできる、運営側の管理画面です（SUPER_ADMIN 権限必須）。

## 6. ライセンスプラン

Miroku には 3 つのプランがあります。

| プラン         | 説明                               | メンバー数 |
| -------------- | ---------------------------------- | ---------- |
| **TRIAL**      | 期間限定の無料お試し（初期 30 日） | 制限あり   |
| **STANDARD**   | 中小企業向け                       | 上限あり   |
| **ENTERPRISE** | 大企業向け、カスタム対応           | 無制限     |

### ライセンス状態の遷移

- 新規テナント作成時、自動で **TRIAL プラン**・**ライセンス開始日＝今日**・**ライセンス終了日＝今日 + 30 日** がセットされ、契約履歴に「TRIAL_START」が残ります。
- スーパー管理者がテナント詳細から、プラン種別・メンバー数上限・期間を変更できます。

### ライセンスの状態（バッジ）

- 緑バッジ — 有効（期限まで 8 日以上）
- 青バッジ — トライアル中
- 黄バッジ — 期限まで 7 日以下（警告）
- 赤バッジ — 期限切れ（データ登録・更新・削除が制限される）

### メンバー数のカウント対象

ライセンス人数にカウントされるのは **「役割 = メンバー (USER)」かつ「入力担当者 (isOperator) ではない」ユーザー** のみ。

- 管理者 (ADMIN): カウント外
- 入力担当者: カウント外
- スーパー管理者: そもそも特定テナントに紐づかない

メンバー編集でロールを変更する際、**カウント外 → カウント内** への遷移時には自動でライセンス上限チェックが行われ、超過する場合はエラーで保存できません。

## 7. 速報機能の仕組み（技術メモ）

非エンジニアの方は読み飛ばして構いませんが、運用上知っておくと便利な内容です。

- 速報通知は **Supabase Realtime broadcast** を介してリアルタイム配信されます。
- ポーリングは廃止済み。アプリのサーバ（AWS Amplify）には常時接続を持たせず、Supabase が WebSocket を肩代わりするため、Lambda 環境の制約と両立します。
- チャネル名は `tenant-events-{テナントID}` で、テナント単位で分離されています。
- イベントは 2 種類:
  - `new-record`: 速報通知（speed up animation 再生）
  - `data-changed`: ダッシュボードのデータ再取得トリガー（debounce 付き）
- 速報用データ（メンバー名・金額・速報メッセージ・動画 ID）は payload には乗せず、認証済み API `/api/sales/breaking-news?id=...` から取得する設計のため、テナント横断の情報漏えいリスクは最小化されています。

**運用設定:**

- 本番運用時は Supabase ダッシュボードの「Database → Publications → supabase_realtime」に `SalesRecord` テーブルを含める設定が必要です。
- アプリの環境変数 `NEXT_PUBLIC_SUPABASE_URL` / `NEXT_PUBLIC_SUPABASE_ANON_KEY` / `SUPABASE_SERVICE_ROLE_KEY` を設定してください。

## 8. よくあるユースケース

### 営業チームでの朝会

ディスプレイモードで **昨日の売上ランキング** や **月次トレンド** を自動表示。朝会で全員が数字を共有できます。

### 売上達成時の盛り上げ

メンバーが売上を登録すると、全画面に **速報動画** が流れてチーム全体で祝福。LINE や Google Chat にも自動通知が飛びます。

### 大型案件の登録

案件種別や契約日をカスタムフィールドで詳細記録。後からレポート画面で、案件種別ごとの売上推移を確認できます。

### 期末の振り返り

レポートビューで **月別トレンド・目標達成予測・営業段階別分析** を確認。次期の戦略立案に活用できます。

### 代理店・事務員による代理入力

営業担当者が外出中でも、事務員（入力担当者）が代理で売上を登録可能。ライセンス人数にはカウントされません。

### 不審なログイン試行の調査

スーパー管理者は `/admin/logs` で全テナントの USER_LOGIN_FAILED ログを確認できます。失敗理由・IP アドレス・試行されたメールアドレスから、攻撃の有無やブルートフォースを早期に検知できます。

## 10. セキュリティ

- **マルチテナント分離** — すべての DB クエリに tenantId 条件が含まれ、テナントごとにデータが完全分離
- **認証** — NextAuth.js による JWT セッション管理。シークレットの長さ・既知の弱い値は本番起動時に検証
- **パスワード暗号化** — bcrypt によるハッシュ化（rounds: 12）
- **タイミング攻撃緩和** — ログイン失敗時もダミー bcrypt 比較を実行し、応答時間からのユーザー存在判定を困難に
- **監査ログ** — すべての主要操作と、ログイン失敗・IP アドレスを記録
- **HTTPS 通信** — 通信の暗号化（HSTS 設定済み）
- **CSP (Content Security Policy)** — XSS 攻撃対策。Supabase Realtime の WebSocket（`wss://*.supabase.co`）を許可リスト化
- **環境変数の保護** — `next.config.ts` のクライアント露出ブロック削除済み。`SUPABASE_SERVICE_ROLE_KEY` は遅延初期化でクライアントに渡らないよう保護
